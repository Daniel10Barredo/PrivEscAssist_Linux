Capabilities

Explicación

Las capacidades en Linux son atributos especiales que se pueden asignar a procesos, binarios, servicios y usuarios para otorgarles privilegios específicos que normalmente estarían reservados para el usuario root. Estos privilegios podrían incluir la capacidad de interceptar tráfico de red o montar/desmontar sistemas de archivos.

Ataque

Existen algunos archivos por defecto que utilizan esta configuración especial, pero si un usuario modifica estas configuraciones erróneamente, podría permitir a un atacante realizar una escalada de privilegios. Las siguientes capacidades son peligrosas y debemos prestarles especial atención:

▪ CAP_SETUID: Permite establecer el ID de usuario efectivo en el ID de usuario real. Es uno de los más interesantes y deberemos aplicar técnicas similares a las vistas en la sección SUID & GUID.

▪ CAP_SETGID: Permite establecer el ID de grupo efectivo en el ID de grupo real. Es uno de los más interesantes y deberemos aplicar técnicas similares a las vistas en la sección SUID & GUID.

▪ CAP_CHOWN: Permite cambiar el propietario y el grupo de cualquier archivo en el sistema.

▪ CAP_DAC_OVERRIDE: Permite ignorar los permisos de acceso a archivos y realizar operaciones de lectura, escritura y ejecución en cualquier archivo.

▪ CAP_DAC_READ_SEARCH: Permite leer y buscar cualquier archivo en el sistema, incluso si los permisos de lectura o búsqueda no están concedidos al usuario.

▪ CAP_NET_RAW: Permite utilizar sockets de red de manera directa, lo que podría permitir realizar acciones de red maliciosas o no autorizadas.

▪ CAP_SYS_ADMIN: Concede una amplia variedad de capacidades de administración del sistema, como montar sistemas de archivos, cargar y descargar módulos de kernel, y realizar operaciones de administración de red.

▪ CAP_SYS_PTRACE: Permite rastrear y manipular los procesos de otros usuarios, lo que podría ser utilizado para obtener información confidencial o realizar ataques.

▪ CAP_SYS_MODULE: Permite cargar y descargar módulos de kernel en el sistema, lo que puede ser explotado para cargar módulos maliciosos o modificar el comportamiento del kernel.

▪ CAP_FOWNER: Permite ignorar los permisos de propiedad de archivo y realizar operaciones de cambio de propiedad en cualquier archivo.

▪ CAP_SETFCAP: Permite establecer atributos de archivo extendidos, lo que podría utilizarse para modificar atributos críticos de seguridad de archivos.

Comprobación

Para identificar todos los binarios que tienen capacidades, utilizaremos el siguiente comando: 

getcap -r / 2>/dev/null
